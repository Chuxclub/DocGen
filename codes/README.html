DocGen - Codes Sources

Auteur(s): Florian Legendre

Que contient ce dossier?

  • DocPageModule/ => Contient toutes les classes permettant de créer la banière (ou header) des pages de documentation et les pages de documentations elles-mêmes (locales et globales)
  • IndexModule/ => Contient toutes les classes permettant de générer des index (aka. table des matières) de types divers. J'en compte pour l'instant deux: les index alphabétiques (les entrées sont triées selon l'ordre alphabétique) et les index hiérarchiques (les entrées sont triées selon leur place dans une arborescence ou un ordre hiérarchique quelconque)
  • MatlabModule/ => Contient les classes (une seule pour l'instant) générales à Matlab et qui permettent de standardiser certains éléments de code que j'utilise beaucoup (comme le clearScript() )
  • StringModule/ => Contient les classes permettant de manipuler des chaînes de caractère plus facilement. Cela inclut des chaînes de caractère très spécifiques comme les chemins absolus ou les noms de fichiers qui nécessitent, après traitement, des appels systèmes

Comment contribuer?

Prérequis à la contribution

Ce projet est versionné sous Git. Toute participation à ce projet implique que vous sachiez utiliser Git. Si besoin, voici un lien contenant des diapositives de formation et un site internet d'exercice: https://github.com/Chuxclub/GitLearn . Votre autoformation devrait vous prendre une dizaine d'heures. A raison d'1 heure par jour vous serez capable de participer à n'importe quel projet versionné et d'utiliser toute la puissance de ce logiciel sur vos propres projets. Vous gagnez en temps et en sécurité!
Ce projet est documenté par DocGen (il s'autodocumente). Il est donc attendu que la documentation soit régulièrement mise à jour par exécution du script DocGenScript.m. Si vous ajoutez des fichiers ou modules au projet il faudra enrichir ce script et mettre à jour la documentation par exécution de ce-dernier.
Ce projet suit une arborescence standardisée expliquée dans son README globale ici. Les fichiers que vous créerez pour ce projet doivent donc être triés selon cette arborescence.
Enfin, les nouvelles fonctionnalités ajoutées à DocGen s'appuient beaucoup sur la fonction Publish de Matlab et notamment sur l'inclusion de balises HTML. Il est donc recommandé la lecture des pages de documentation suivantes:
  • Publish and Share Matlab Code
  • Page de manuel de la fonction Publish de Matlab
  • Matlab Publish Markup
  • Cours accéléré sur le langage de description HTML créé par SoloLearn

Exigences concernant les contributions

Ce projet suit le paradigme orienté objet. Une maîtrise experte de ce paradigme n'est pas nécessaire pour participer mais une connaissance suffisante est tout de même attendue. Quelques conventions de nommage:
  • maFonction
  • maVariable
  • MA_CONSTANTE
  • MaClasse

Par où commencer?

Pour découvrir le code:
  1. Aller dans codes/ puis ouvrir DocGen.m
  2. Suivre le chemin de construction d'une notice locale => vous passerez par les fichiers IndexModule/IndexA.m, DocPageModule/Manual.m, etc. en autres choses...
  3. Faire de même avec une notice globale
Pour commencer à contribuer (la difficulté/connaissance du code estimée est notée avec des étoiles *** sur 5):
  • ** Clarifier les entrées sorties de chaque méthode (quelles structures de données en entrée? En sortie?)
  • *** Modifier le code pour le rendre plus modulaire
  • ** Tester la résistance du code aux crashs/bugs et débugger en conséquence
  • **** Implanter un index alphabétique de toutes les fonctions du projet accessible par un bouton sous le header
  • ***** Implanter un système ou plugin permettant de générer des graphes d'appels de fonctions qui s'afficheront dans les notices générées
